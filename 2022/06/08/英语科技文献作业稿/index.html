英语科技文献综述稿

Hello, everyone. Today I will introduce the Use of the Hough Transformation to Detect Lines and Curves in Pictures with the theme of computer vision.

It is mainly divided into two parts, the first thing is the background and then is the fundamentals of Hough Transformation.

As you can see the Hough transformation was registered as a patent in 1962 and widely used because of its performance. So the main content I’m going to talk about is the fundamentals to detect lines and curves with it.

We can easily draw a line in Polar coordinate system which is composed of a set of points but it’s hard for computer to determine which set in an image is corresponding to a line. We should find an easier and clearer standard.

For a given line, we can determine specific ρ and θ. Then, the equation is satisfied for each x, y point belonging to this line. So, we can draw the mapping-point in Hough space. In a similar way, we can transform a point in Polar coordinate system to a sinusoid in Hough space.

So you can find that if we draw points which form a line in the image space, we will obtain a bunch of sinusoids in the Hough space. And, they are intersecting at exactly one point, and that point is the line in Polar coordinate system (image space).

It’s prone to be a line if many curves intersect at a point.

Thus, the problem of detecting collinear points can be converted to the problem of finding concurrent curves.

What we actually do is voting the (ρ,θ) values. We continue this process for every point on the line. After the process, we will get the table of the (ρ,θ) and its votes.

We can give a threshold, if the number of concurrent curves is larger than it, we can say that we find a line in the image space.

It’s easy to infer that the larger threshold we use, the fewer lines we will find.

And for curves curves such as circle, the fundamentals is similar, and you just need to change the number of parameters.
